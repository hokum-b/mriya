window states

  • normal: tiled on the strip.
  • maximized: fills the strip height; width is set by the user.
  • floating: free-floating window not bound to the strip.
  • fullscreen: covers the entire monitor.

keybinds

  • mod + return open terminal
  • mod + d open dmenu
  • mod + w / mod + shift + q close focused window
  • mod + shift + e quit wm
  • mod + shift + r restart wm
  • mod + h/j/k/l or left/right focus left/right
  • mod + up workspace up
  • mod + down workspace down
  • mod + 1-9 switch to workspace
  • mod + shift + 1-9 move window to workspace
  • mod + ctrl + 1-9 toggle workspace view
  • mod + ctrl + shift + 1-9 toggle workspace on window
  • mod + space zoom (move to master)
  • mod + shift + space toggle floating
  • mod + f toggle maximize
  • mod + shift + f toggle fullscreen
  • mod + tab switch to last workspace
  • mod + shift + j/k decrease/increase gaps
  • mod + ctrl + j reset gaps
  • media keys: volume and brightness

customizing

edit the keys array in src/config.h (lines 118-165). each entry has four fields:

  • mod -- modifier mask. use modkey for super, modkey|shiftmask for super + shift, modkey|controlmask for super + ctrl, or 0 for unmodified. the cleamask macro (line 102) strips numlock and capslock.
  • keysym -- x11 keysym constant. common: xk_return, xk_space, xk_tab, xk_h/xk_j/xk_k/xk_l, xk_f, xk_d, xk_w, xk_e, xk_r, xk_1-xk_9, xk_up/xk_down/xk_left/xk_right. media keys use xf86x_audiomute, xf86x_audiolowervolume, xf86x_audioraisevolume, xf86x_monbrightnessdown, xf86x_monbrightnessup.
  • func -- callback pointer. available: spawn, killclient, quit, restartwm, focusleft, focusright, ws_up, ws_down, setgaps, zoom, togglefloating, togglemaximize, togglefullscreen, view, toggleview, tag, toggletag, tagmon, movemouse, resizemouse.
  • arg -- string passed to the callback. for spawn it is the shell command (use term, dmenu defines). for tag/view it is the workspace index as a digit ("0"-"8"). for setgaps it is "+2", "-2" or "0". pass null for callbacks that ignore the argument.

the tagkeys macro (lines 87-91) generates four bindings per workspace key: view (switch), toggleview (toggle), tag (move), toggletag (toggle move). example for xk_1 with tag 0:

{ modkey, xk_1, view, "0" },
{ modkey|controlmask, xk_1, toggleview, "0" },
{ modkey|shiftmask, xk_1, tag, "0" },
{ modkey|controlmask|shiftmask, xk_1, toggletag, "0" }.

the key macro (line 94) is a shorthand: key(mod, keysym, func, arg) expands to { mod, keysym, func, arg },. callback resolution in keypress() (lines 1647-1655) uses xkbkeycodetokeysym() then a linear scan of the keys array.

mouse

  • mod + left drag move window (tiled: reorder/scroll; floating: move)
  • mod + right drag resize window
  • click title bar close button kill client
  • click title bar maximize button toggle maximize

customizing

edit the buttons array in src/config.h (lines 168-171). fields: mod (modifier mask), button (x11 constant: button1 left, button3 right), func (movemouse or resizemouse), arg (unused, pass null). event handling in buttonpress() (lines 1507-1553) checks title bar clicks first, then iterates the buttons array at lines 1548-1551 using cleamask.

title bar

windows without a frame (panels, dialogs) get no title bar. the title bar shows the window title and optional close/maximize buttons. the border has two layers: outer and inner, with separate colors for focused and unfocused states.

customizing

three defines in src/config.h control the title bar:

  • show_titlebar (line 30): set to 0 to suppress frames and title bars entirely. the frame is created in manage() at line 982 when this is 1 and the window is not a panel or dialog.
  • show_title (line 31): set to 0 to hide the title text. drawn at lines 1475-1480 via xdrawstring(), truncated to 60 chars.
  • show_buttons (line 32): set to 1 for close (x) and maximize (square) buttons on the right. close calls killclient(), maximize calls togglemaximize(). hit detection in buttonpress() at lines 1518-1532.

window rules

rules match by class, instance or title. they set initial workspace, floating state or target monitor.

customizing

edit the rules array in src/config.h (lines 72-77). each rule entry has fields:

  • class -- match against wm_class (e.g. "gimp", "firefox").
  • instance -- match against wm_instance_name (res_name).
  • title -- match against wm_name (window title).
  • tags -- bitmask of target workspaces (e.g. 1 << 8 for workspace 9).
  • isfloating -- set to 1 to start floating.
  • monitor -- target monitor index, -1 for current.

the first matching rule wins. any field set to null is a wildcard. example default entries:

{ "gimp", null, null, 0, 1, -1 } → gimp starts floating
{ "firefox", null, null, 1 << 8, 0, -1 } → firefox on workspace 9

layouts

  • [=] strip (horizontal tile)
  • [m] monocle (windows stacked full-size)

the layouts array (lines 80-84) maps a display symbol to a layout function. the second entry { "><>", null } has a null function pointer as a placeholder. tile() (line 657) is an alias for arrange(). monocle() (lines 661-674) resizes every visible client to fill the full monitor area minus gaps and title bar.

gaps and snap

  • inner_gap (default 15): pixels between adjacent windows. defined at line 10. the global inner_gaps (line 116) is initialized from this and adjusted at runtime by setgaps() (lines 823-830): "0" resets, "-" decrements by 2, "+" increments by 2. clamps at 0.
  • outer_gap (default 50): pixels from screen edge to outermost windows. defined at line 11. adjusted together with inner_gap at runtime.
  • snap (default 32): distance for floating windows to snap to screen edge during mouse move. defined at line 12. checked in movemouse() at lines 1309-1314.

double border

  • outer_border_width (default 0): outer ring thickness. line 5.
  • inner_border_width (default 0): inner ring thickness. line 6.
  • border_width: sum of both. line 7.

drawn in drawtitle() at lines 1495-1503. outer ring uses col_norm_outer_border / col_sel_outer_border, inner uses col_norm_inner_border / col_sel_inner_border. each ring is a series of nested xdrawrectangle() calls.

autostart

the autostart_cmds array in src/config.h (lines 55-64) lists shell commands run at startup via /bin/sh -c in autostart() (lines 304-313).

xsetroot -solid '#222222' → solid background
xrdb -merge ~/.xresources → load x resources
picom --backend glx → compositor
dunst → notification daemon
sheet --restore → restore terminals
polybar example → status bar
nm-applet → network tray
volumeicon → volume tray

autostart_len (line 66) is computed via sizeof. children that fail exec call _exit(1) silently.

workspace tags

the tags array (line 69) holds 9 string labels defaulting to "1" through "9". published via ewmh for bars and pagers. map to internal indices 0-8 used by view(), tag(), toggletag() and toggleview(). max_workspaces (hardcoded to 10 at mriya.c line 21) limits the range.

misc settings

  • insert_end (line 174): 1 appends new clients to the list end, 0 prepends. affects attach() (lines 927-944).
  • strip_align (line 176): 0 = left, 1 = right, 2 = center when strip is narrower than screen. enforced in arrange() (lines 608-614) and ensure_visible() (lines 879-885).
  • scroll_step (hardcoded at line 23): pixels per scroll key press (default 300).
  • title_height (hardcoded at line 24): title bar height in pixels (default 20).
  • button_size (hardcoded at line 25): title bar button size (default 12).
  • max_clients (hardcoded at line 20): max managed clients (default 256).
  • modkey (line 35): modifier mask. default mod4mask (super), change to mod1mask for alt.
  • mousemask (line 37): event mask for mouse drag: buttonpressmask|buttonreleasemask|pointermotionmask.

rebuild cycle

make clean && make → recompiles
mod + shift + r → re-execs binary in-place

restartwm() (lines 1980-1983) sets restart = 1 and running = 0. run() exits the event loop, cleanup() frees resources and closes the display. main() (line 2000) calls execvp(argv[0], argv) to run the new binary.

wm won't start

checkotherwm() (lines 1711-1717) tries to select substructureredirectmask on the root window. if another wm holds it, xerrorstart() calls die("mriya: another window manager is already running"). from a tty run ps aux | grep -i xinit to find and kill the session, or use startx -- :1 for a different display.

blank screen on start

drawbar() (line 1463) only calls xclearwindow(dpy, root). there is no built-in wallpaper, bar or compositor. if autostart_cmds is empty or programs are missing, the screen stays blank. verify each command independently. children that fail exec call _exit(1) silently.

keys unresponsive

grabkeys() (lines 563-573) calls xgrabskey() on the root window. if another program grabs the same combo first, mriya's grab fails silently. run xev to check keysyms. modkey (default mod4mask) must match your physical modifier. some hardware maps super to mod4, others to mod1 or mod3. updatenumlockmask() (lines 550-561) excludes num_lock from modifier matching.

window not visible

each client has a workspace field. if it differs from selmon->workspace, isvisible(c) (line 103) returns false and arrange() moves the window off-screen (lines 627-630: x = width * -2). switch workspaces with mod + number. override_redirect windows are skipped entirely in maprequest() (lines 1663-1669). transient windows (dialogs) are set to floating in manage() (lines 978-980).

title bar absent

in manage() (line 982), a frame is only created when show_titlebar is 1 and the window is not a panel, dock, toolbar, menu, splash, notification or utility. types detected by updatewindowtype() (lines 404-421) via _net_wm_window_type. dialogs are floating but still get a frame if show_titlebar is 1.

not in display manager

display managers read /usr/share/xsessions/*.desktop. install.sh (lines 6-11) writes it. if missing, create:

[desktop entry]
name=mriya
comment=scrolling x11 window manager
exec=/usr/bin/mriya
type=application
desktopnames=mriya

install.sh uses /usr/bin/mriya; make install uses /usr/local/bin/mriya.

gaps reset on restart

setgaps() (lines 823-830) modifies global inner_gaps and outer_gaps at runtime. these are not persisted. after mod + shift + r, values revert to the defines. copy desired values into config.h and recompile.

strip won't scroll

get_total_strip_width() (lines 575-594) sums all visible client widths plus gaps. if the total is less than the monitor width, scroll_x is forced to 0. open more windows or maximize one (mod + f) to exceed the screen edge. scroll step is 300px (scroll_step, line 23). left limit is 0; right limit is -(total - screen_width).

reorder by drag fails

movemouse() (lines 1270-1385) reorders tiled (non-floating, non-fullscreen) windows. the drag must exceed snap (default 32px) to initiate. past 1/4 of window width (win_w / 4, line 1344) swaps with the adjacent client by rewiring the linked list. floating or fullscreen windows skip the reorder code entirely.

floating clips off-screen

in movemouse(), snap-to-edge checks (lines 1309-1314) clamp nx/ny to monitor bounds minus outer_gaps. if snap (default 32) is too low, a window can be dragged partway off before snapping. increase snap in config.h. there is no clamp beyond the snap check, so a fast drag can push windows entirely off-screen.

reading stderr

run mriya from a tty: startx ./mriya 2> ~/mriya.log. die() (lines 264-276) writes to stderr with vfprintf. if the format string ends with ':', perror() output is appended. x error messages are printed by xerror() (lines 1724-1732) for codes other than badwindow, badmatch, etc.

restart wm

mod + shift + r calls restartwm() which sets restart = 1 and running = 0. the event loop exits, cleanup() runs, main() (line 2000) calls execvp(argv[0], argv). if execvp fails, die("mriya: failed to restart") is printed.

force quit

switch to a tty with ctrl + alt + f2. pkill mriya or pkill -9 mriya if sigterm is ignored. pkill xinit terminates the entire x session.

debug key events

run xev | grep -a2 keysym, press the key, note the output. keypress() (line 1651) uses xkbkeycodetokeysym() which respects the active xkb layout. compare against keysym values in the keys array. if the modifier does not match, adjust modkey or use xmodmap.

ewmh support

mriya sets _net_client_list, _net_active_window, _net_number_of_desktops, _net_current_desktop, _net_desktop_names, _net_wm_desktop, _net_wm_window_type, _net_wm_state and _net_supported. also provides _net_supporting_wm_check.

signals

sigchld handler reaps zombie children from spawned processes.

platforms

tested on: generic linux x86_64, freebsd 15.1, netbsd 10.1, openindiana hipster 2026.04.
